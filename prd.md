## 一、 产品命名与定位

- **产品名称：** **Salary Shield**
- **核心目标：** 为低数字化水平的劳动者提供从“心理博弈”到“法律存证”的一站式支持。

## 三、 具体功能模块说明

## 三、 具体功能模块说明

### 1. 沟通博弈模块 (The Negotiator)

- **输入：** **优先支持按住说话（语音录入）**，同时也支持文字输入/粘贴。
- **逻辑：** 
    - **分析：** AI 分析老板的借口（如：哭穷、推诿、威胁）。
    - **生成：** 提供 3 种维度的回复方案：
        - *情理说服型：* 针对“熟人社会”，保持面子但点明困难。
        - *坚定立场型：* 明确还款期限，告知后续行动。
        - *法律预警型：* 引用具体法规进行威慑。
- **技术点：** Prompt Engineering。设定 AI 为“拥有 15 年经验、极具同理心的基层法律协调员”。调用语音识别 API (如 step-asr) 实现语音转文字。

### 2. 证据链与告知书模块 (The Archivist)

- **输入：** 批量上传微信聊天截图、转账记录、语音转文字记录。
- **处理流程：**
    1. **多模态解析：** 调用 **Qwen-VL**（通义千问视觉大模型），它能直接理解截图中的对话逻辑，不仅提取文字，还能识别“谁在说话”以及“关键承诺时间”。
    2. **要素提取：** 自动整理出：欠款主体、入职时间、欠款金额、最后承诺日期。
    3. **PDF 生成：** 使用 Python 库（如 `FPDF` 或 `ReportLab`）将上述要素填入**标准版**“限期支付告知书”模板。
- **输出：** 一份标准格式、带有法律威慑力的 PDF 文件，用户可直接在微信发送给老板。

### 3. 公益导航与模拟律师模块 (The Lighthouse)

- **功能分层：**
    1. **维权路径导航：** 基于用户当前情况，给出具体的线下维权路径。（如石家庄当地的劳动监察大队地址、电话）。
    2. **模拟律师咨询（AI Lawyer）：** 
        - **核心创新：维权案卷包 (Case File)**。系统会自动打包用户之前上传的聊天记录、证据截图、AI 分析结果以及生成的《告知书》。
        - **交互体验：** 用户无需重复介绍案情。AI 律师读取“案卷包”后，直接基于完整上下文进行咨询对话。
        - **场景：** 模拟与公益律师的线上沟通，提供更深度的法律建议或行动指导。

- **技术点：** 
    - **轻量级 RAG/System Prompt：** 存储法条和机构信息。
    - **Context Management：** 将前两个模块生成的结构化数据（Case File）作为 System Context 传递给对话模型，实现“降本增效”的无缝咨询体验。

## 四、 技术线路 (Tech Stack)

| **维度** | **选型** | **理由** |
| --- | --- | --- |
| **前端框架** | **Streamlit** | 开发极快，自带 UI 组件，适合 Hackathon 快速产出。 |

# 五、其他说明

## 知识库的具体说明（在不实用数据库和向量分析的时候模拟类似效果）

### 方案 A：直接作为“系统提示词（System Prompt）” —— 最推荐

如果你的核心知识库（比如《劳动法》精华）只有几千字，**最轻量、最稳健**的方法是：**不搞 RAG，直接把知识喂给 Prompt。**

- **原理：** 现在 Qwen-Max 等大模型的上下文窗口（Context Window）非常大（通常 32k 到 128k 甚至更多）。你可以直接把最核心的维权逻辑写在代码里的一个长字符串中。
- **优点：** 零延迟，零报错风险，逻辑最简单。
- **适用：** 核心规则、常用话术模板。

### 方案 B：本地 Markdown/JSON 关键词匹配

如果你有几十个维权案例或不同地区的法律条文。

- **原理：** 将知识存为一个 `legal_knowledge.json` 文件。用户提问时，先用 Python 进行简单的关键词提取，从 JSON 里找出相关的段落，再塞给 LLM。
- **优点：** 结构清晰，方便你随时修改知识库内容。
- **适用：** 存储石家庄各区劳动监察大队的地址和电话。
